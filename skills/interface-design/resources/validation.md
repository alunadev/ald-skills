# Validation & System Management

## The Mandate Check
Before presenting:
"If the user said this lacks craft, what would they mean?" -> Fix it first.

## Self-Audit Checklist
- [ ] **Swap Test**: Would the design feel the same with Inter/Arial?
- [ ] **Squint Test**: Is hierarchy quiet but perceivable?
- [ ] **Signature Test**: Can you point to 5 instances of the signature element?
- [ ] **Token Test**: Do your CSS variables read like this product's domain?

## Memory & Consistency
- Persist patterns in `.interface-design/system.md`.
- Read system state before any new interface task.
- Ensure transitions between light/dark themes keep the same "intent" constraints.
